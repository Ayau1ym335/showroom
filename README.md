# Dori — сайт мультибрендового шоурума

Next.js 15 (App Router) + TypeScript + Tailwind CSS + Supabase.

## Запуск локально

### 1. Установка зависимостей
```bash
npm install
```

### 2. Настройка Supabase
1. Создайте проект на [supabase.com](https://supabase.com) (бесплатного тарифа достаточно для старта)
2. Откройте **SQL Editor** и выполните целиком файл `supabase/schema.sql` — он создаст все таблицы, политики безопасности и стартовые категории
3. Откройте **Storage** и убедитесь, что бакеты `product-media` и `banners` созданы (схема создаёт их автоматически, но можно проверить)
4. Откройте **Authentication → Users** и создайте одного пользователя вручную — это будет владелец магазина, вход в админку идёт через email + пароль этого пользователя

### 3. Переменные окружения
Скопируйте `.env.example` в `.env.local` и заполните:
```bash
cp .env.example .env.local
```
- `NEXT_PUBLIC_SUPABASE_URL` и `NEXT_PUBLIC_SUPABASE_ANON_KEY` — из Project Settings → API
- `NEXT_PUBLIC_ADMIN_EMAIL` — email пользователя, созданного в шаге 2.4

### 4. Запуск
```bash
npm run dev
```
Сайт откроется на `http://localhost:3000`, админка — на `http://localhost:3000/admin/login`.

## Структура проекта

```
app/
  page.tsx                    — главная страница-лендинг (8 секций)
  catalog/page.tsx             — каталог с фильтрами
  product/[slug]/page.tsx      — страница товара
  admin/
    login/                     — вход в админку
    page.tsx                   — дашборд
    products/                  — список, добавление, редактирование товаров
    categories/                — категории и бренды
    reviews/                   — модерация отзывов
    settings/                  — контакты и ссылки

components/
  layout/                      — шапка, подвал, кнопка WhatsApp
  catalog/                     — карточка товара, фильтры
  product/                     — галерея, выбор размера
  admin/                       — форма добавления товара

lib/
  supabase/                    — клиенты для браузера и сервера
  data.ts                      — все запросы к базе данных
  utils.ts                     — форматирование цены, ссылка WhatsApp

types/index.ts                 — все TypeScript-типы

supabase/schema.sql             — полная схема базы данных
```

## Наполнение сайта данными

После разворачивания у вас будет пустой каталог. Есть два пути:

1. **Через админку** (рекомендуется) — зайдите в `/admin/products/new` и добавляйте товары так же, как вы публикуете посты в Instagram: фото → название → цена → бренд → готово.
2. **Массово через SQL** — если у вас уже есть таблица товаров (Excel/CSV), можно один раз загрузить их через SQL Editor в Supabase — обращайтесь, если нужна помощь с преобразованием файла в SQL-вставки.

Не забудьте создать записи брендов (`brands`) до того, как начнёте добавлять товары — иначе выпадающий список в форме будет пустым.

## Деплой

Рекомендуемый хостинг — [Vercel](https://vercel.com) (тот же принцип, что у Next.js). Подключаете GitHub-репозиторий, добавляете те же переменные окружения из `.env.local` в настройках проекта Vercel — деплой автоматический при каждом пуше.
